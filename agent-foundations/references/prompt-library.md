# Memory System Prompt Library / 系統建構 Prompt 庫

Copy-paste prompts to have an AI agent build or maintain a memory system.
Fill in the bracketed placeholders before sending.

> 直接複製貼上，把 `[...]` 換成你的內容。

## Prompt 1 — Reorganize existing files / 現有檔案結構重組

```text
想把下面這些檔案整理成 AI Agent 的記憶系統，請你幫忙做三件事：
1. 把檔案分成三類：
   - L1（每次對話自動載入，例如個人偏好、當前工作）
   - L2（按主題載入，例如特定專案、特定知識領域）
   - L3（歷史存檔，平常不載入）
2. 找出明顯的問題：
   - 有沒有重複的資訊放在不同檔案
   - 有沒有檔案太長需要拆分（超過200行）
   - 有沒有檔案內容太雜應該分成多個檔案
3. 給出一個推薦的新檔案結構，包含：
   - 每個檔案建議的路徑和名稱
   - 每個檔案應該放什麼內容
   - 哪些舊檔案要合併或拆分

檔案清單：
[請在此貼上目錄 tree 或 ls -la 輸出]

使用情境：
[例如：日常工作助理、寫作夥伴、程式碼 reviewer]
```

## Prompt 2 — Plan memory tiers from scratch / 從零規劃記憶分層

```text
要為 AI Agent 設計一套記憶系統，請你根據以下情境，給出一個三層的記憶架構。

情境：
- 身份：[例如：全端工程師]
- 主要任務：[例如：專案開發、架構規劃]
- 每天會跟 agent 討論的事：[列三到五項]
- 偏好的互動方式：[例如：繁體中文、直接給 Code、不要廢話]
- Context window 限制：[例如: 1M token]

請設計：
1. L0（索引層）：一份 MEMORY.md 的內容，列出所有檔案的路徑和用途。
2. L1（工作記憶）：推薦 2-3 個檔案，各自負責什麼、預估幾行。
3. L2（主題記憶）：推薦 4-8 個主題檔案，各自放什麼內容、觸發載入條件。

請給出每個檔案的路徑、預估行數、包含的內容（條列式）與更新頻率。
最後幫我試算總佔用 Token 是否有壓在 Context 的 20% 以內。
```

## Prompt 3 — Build a brain.md working-memory template / 建立 brain.md 範本

```text
請設計一份 brain.md 範本，這是 AI Agent 每次對話必讀的「當下工作記憶」。

工作模式：
- 同時進行的專案數：[例如：3 個]
- 每天切換專案次數：[例如：5 次]
- 常卡住的原因：[例如：API 沒通、客戶未回覆]
- 希望 agent 看完能做什麼：[例如：立刻接續上次開發進度、知道哪些票正在等]

請包含以下區塊，並控制在 80 行以內（請用實際範例填入，語氣要口語且簡短）：
1. 檔頭註解（用途與更新規則）
2. 今天的焦點 (1-2 件大事)
3. 進行中專案進度
4. 等回覆 / 被 Block 的事項
5. 卡住的技術難點及嘗試方向
6. 今天的小事 (15 分鐘可解決)
7. 底部更新時間戳記
```
